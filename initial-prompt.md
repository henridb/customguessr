I want to build a web app, kinda similar to geoguessr but without the streetview part. For the technologies, I want to use typescript with pnpm, I want to build with bun, and I want to use tailwind for the theming. The map will be a single image to be determined later. The map represents a galaxy, and the idea is that a game is composed of 5 rounds, in each round, the game prompts the player with the name of a planet or star, and the player has to click somewhere on the map. Each round is worth 5000 points, for a maximum total of 25000 points for a game, the player is awarder the full 5000 points if they clic exactly on the planet, and otherwise it decays proportionaly to the inverse of the distance (the exact fomula will be tweaked later for a good difficulty balance). I only want two pages, the admin page is used to easily populate the planets and stars database (if the admin clics somewhere empty, a modal to create a new planet at this location appears, if the admin clicks on a planet or star, a modal to modify the existing planet or star appears), and a player page (the home page), with only a new game button, and clicking this button will launch the player in a game. Everything should be client side for simplicity's sake, this means that the admin page will not directly change the database, but rather will allow the admin to export a json file witch will be used to replace the one in this repo. This means that the security of the admin page is not critical, so it can simply be done client side, with a hardcoded password (potentialy with some obfuscation to make cheating slightly harder).
